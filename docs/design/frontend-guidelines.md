# 悟道 (Wudao) 前端开发与 UI/UX 规范

本文件定义了悟道项目前端开发的视觉、交互及工程准则。所有前端变更必须严格遵循此规范，以保持系统高度一致的“类 Apple”原生体验。

## 1. 视觉设计规范 (Apple Aesthetic)

*   **圆角体系**：统一使用 `rounded-apple` 系列。
    *   `lg`: 12px (标准按钮、小卡片)
    *   `xl`: 16px (对话框、中型卡片)
    *   `2xl`: 24px (大型容器、输入框区域)
*   **材质与层次**：
    *   **Apple Glass (毛玻璃)**：使用 `backdrop-blur-apple`。背景色应根据模式在 `white/70` 到 `black/60` 之间切换，配合 `border-black/5` 的极细边框。
    *   **Apple Card (卡片)**：背景使用纯色或毛玻璃，浅色模式下需具备 `shadow-apple-card` 阴影。深色模式下应减弱阴影，增强 `border-white/10` 边框感。
*   **色彩与对比度**：
    *   **主色调**：使用 `apple-blue` (#007AFF) 作为交互引导色。
    *   **选中态**：在暗黑模式下，选中项背景必须显著（推荐 `bg-apple-blue`），文字颜色强制设为 `white`，禁止使用低对比度的深灰色。
*   **全局样式**：
    *   **文本选中**：统一 `::selection` 为 `bg-apple-blue/30` 和高对比度文字色。
    *   **滚动条**：使用自定义纤细滚动条（见 `index.css`），非操作状态下应保持低调。

## 2. 交互与动效规范 (Interaction & Motion)

*   **加载感知**：
    *   **统一加载态**：列表或面板默认使用 `LoadingIndicator` 与稳定壳层占位，避免白屏和布局跳动。
    *   **谨慎使用 Skeleton**：当前项目已移除全站 pulse Skeleton 体系；只有在某个局部内容确实需要拟态占位时，才为该区域单独设计轻量骨架，禁止恢复通用鱼骨页套路。
*   **响应速度**：
    *   **瞬时切换**：视图（Tab）切换应追求零延迟感知，除非必要，否则应移除位移/淡入淡出等过渡动画。
    *   **物理手感**：细节交互（如悬停、微调）使用 `framer-motion` 的 `spring` 配置，参数参考 `stiffness: 300, damping: 30`。
*   **防形变原则**：
    *   在处理布局高度变化（如输入框升高）时，**禁止使用 `layout` 缩放动画**。必须通过 CSS `transition` 改变 `height`，确保文字、图标不发生任何比例拉伸。

## 3. 组件开发准则

*   **HeroUI 基线**：
    *   新增或重构通用控件时，优先基于 HeroUI v3 React 组件封装项目级组件；不要在业务组件中直接散落大量一次性按钮、输入框和浮层样式。
    *   HeroUI 组件必须通过项目包装层继承 Wudao 的 Apple glass token、圆角、暗黑模式和 i18n 约束；不得引入另一套明显不同的设计语言。
    *   纯图标操作优先使用 `WudaoIconButton` 并提供 `tooltip` 与 `aria-label`，避免只依赖浏览器原生 `title`。
    *   任务工作台、终端和 Runner 等尺寸敏感区域迁移时，必须保持既有固定高度、固定宽度和测量容器不变。
*   **输入框 (Textarea)**：
    *   **标准高度**：单行默认高度固定为 `38px`。
    *   **动态伸缩**：随内容自动升高，当前任务聊天输入框上限约为 `254px`（约 10 行），超出后自动启用内置滚动。
    *   **按钮对齐**：单行时按钮垂直居中；多行时按钮平滑移至右下角并保留固定 Padding。
*   **下拉框 / 浮层菜单 (Dropdown)**：
    *   **触发器统一**：优先复用紧凑胶囊触发器样式，保持 `h-8`、小字号大写标签、轻 hover 反馈，与任务类型/优先级/模型选择器保持一致。
    *   **菜单材质统一**：默认复用 `apple-dropdown`，不要为单个菜单额外叠加更重的玻璃面或多层阴影，避免出现“菜单比页面更重”的割裂感。
    *   **层级原则**：如果触发器位于带 `backdrop-filter` 的父容器中，菜单必须脱离该毛玻璃父层渲染，作为同级独立浮层或 portal 渲染；禁止把需要毛玻璃的菜单直接嵌在另一个毛玻璃容器内部。
    *   **动效原则**：展开/收起动画应控制在 100-140ms，优先使用轻量的 opacity + 轻微位移，scale 变化只允许极小幅度（例如 `0.985 -> 1`），禁止使用会造成“明显等待感”的重动画。
    *   **交互兜底**：菜单展开时必须提供 `fixed inset-0` 的点击空白关闭层，保证关闭路径稳定、点击热区明确。
*   **组件自研原则**：
    *   当第三方库（如 DatePicker）无法深度适配 `apple-glass` 风格或在暗黑模式下表现不佳时，**优先手写轻量级自定义组件**。
    *   自定义组件应复用 `date-fns` 逻辑和项目基础 UI 类名。

## 4. 国际化 (i18n) 规范

*   **字符串管理**：代码中严禁硬编码中/英文字符串。所有可见文本必须存放在 `locales/*.json` 中。
*   **语义化键名**：键名应反映 UI 角色而非字面直译（例如使用 `tasks.agent_chat` 而非 `tasks.workspace_title`）。
*   **动态本地化**：日期、时间、月份等必须根据 `i18n.language` 动态传入对应的 `date-fns` locale 实例。

## 5. 导航与布局

*   **三段式导航**：顶部导航栏必须保持“左 Logo、中菜单、右工具”的布局。
*   **居中原则**：菜单导航 (`nav`) 应使用 `absolute left-1/2 -translate-x-1/2` 确保在任何分辨率下绝对居中。
*   **溢出处理**：处理 Tab 滑块动画时，应暂时移除容器的 `overflow-hidden` 以允许滑块跨越间距。
